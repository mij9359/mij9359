## 🎯 문제 정의

### 상황: 통합 심사결과 조회

```
요구사항:
팀 A의 심사 결과:
├─ 2025년 도전기업
│  ├─ 서류: O (진출)
│  ├─ 발표: 합격
│  └─ 최종: 합격 ✅
│
└─ 2026년 펠로우
   ├─ 서류: O (진출)
   ├─ 발표: 불합격 ❌
   └─ 최종: (없음)

결과: 팀별 × 트랙별 × 단계별 3-state (합격/불합격/없음)
```

### 복잡성

```
❌ 문제 1: DB에서 직접 계산 불가
   ├─ stage_choice 테이블 (합격 팀 목록)
   ├─ track별로 stage 구성이 다름
   └─ 모든 경우의 수를 SQL로? 불가능

❌ 문제 2: N+1 쿼리
   ├─ 팀 500개 × 트랙 5개 = 2500팀
   ├─ 팀마다 각 트랙의 stage_choice 조회
   ├─ 최소 2500 쿼리
   └─ 실제 단계별로 3000+ 쿼리 ❌

❌ 문제 3: 합격/불합격/없음 구분
   ├─ NULL (해당 단계 없음) vs "불합격" (떨어짐)
   ├─ DB에서 구분 복잡
   └─ 서비스 로직에서 구분 필요
```

## 💡 솔루션: 트랙별 집계 + Set 기반 O(1) 조회

```
[Step 1] 전체 트랙의 Stage 메타 로드 (1회)
└─ allStageInfoList = [Track1-St1, Track1-St2, Track2-St1, ...]

[Step 2] 각 트랙별로 필터 (메모리)
└─ Track1의 Stage만: [St1, St2]
└─ Track2의 Stage만: [St1]

[Step 3] 각 Stage별 "합격팀" Set 구축 (트랙별로)
├─ Track1-St1 (서류): {팀1, 팀2, 팀5, ...} → Set<Long>
├─ Track1-St2 (발표): {팀1, 팀3, ...} → Set<Long>
└─ Track2-St1 (서류): {팀1, 팀4, ...} → Set<Long>

[Step 4] 팀×단계×트랙 2중/3중 루프에서 Set.contains 조회
├─ 팀A, Track1, 서류: Set.contains(팀A) → true → "O"
├─ 팀A, Track1, 발표: Set.contains(팀A) → true → "합격"
└─ 팀A, Track2, 서류: Set.contains(팀A) → false → "X"

→ 쿼리: 트랙 × 3단계 = 15쿼리
→ 조회: 팀 × 트랙 × 단계 = O(1) × O(1) × O(1)
```

## 🔥 핵심 구현 코드

```java
private Map<Long, EvaluationResultVO> getEvaluationResultMapByTeam(
        List<Long> teamNos, 
        List<Long> trackNos) {

    Map<Long, EvaluationResultVO> resultMap = new HashMap<>();

    // ──────────────────────────────────────────────────────
    // [Step 1] 전체 트랙의 Stage 메타 한 번에 로드 (N+1 제거)
    // ──────────────────────────────────────────────────────
    
    List<StageInfoByTrackVO> allStageInfoList = 
        pointStageQueryMapper.getStageInfoByTrack();
    
    log.info("[Evaluation] Stage 메타 로드 완료: {} 단계", allStageInfoList.size());

    // ──────────────────────────────────────────────────────
    // [Step 2] 각 트랙별로 처리
    // ──────────────────────────────────────────────────────
    
    for (Long trackNo : trackNos) {
        
        // [2-1] 이 트랙의 Stage만 필터 (메모리)
        List<StageInfoByTrackVO> stageInfoList = allStageInfoList.stream()
            .filter(s -> s.getTrackNo().equals(trackNo))
            .toList();
        
        log.info("[Evaluation] Track {}: {} 단계", trackNo, stageInfoList.size());
        
        // ──────────────────────────────────────────────────────
        // [Step 3] 각 Stage별로 "합격팀 Set" 구축
        // ──────────────────────────────────────────────────────
        
        // 3개 단계 기준: 0=서류, 1=발표, 2=최종
        List<Set<Long>> choiceTeamSets = new ArrayList<>();
        
        for (StageInfoByTrackVO stageInfo : stageInfoList) {
            try {
                // 해당 Stage에 합격한 팀 목록 조회
                TrackAndPagingVO param = TrackAndPagingVO.builder()
                    .trackNo(trackNo)
                    .stageNo(stageInfo.getStageNo())
                    .build();
                
                List<GetU300ChoiceTeamResVO> choiceList = 
                    stageQueryService.getU300ChoiceTeamList(param);
                
                // 팀 번호만 추출해서 Set으로 변환 (O(1) lookup)
                Set<Long> choiceTeamSet = (choiceList != null && !choiceList.isEmpty())
                    ? choiceList.stream()
                        .map(GetU300ChoiceTeamResVO::getTeamNo)
                        .collect(Collectors.toSet())
                    : null;  // null = "이 단계 없음"
                
                choiceTeamSets.add(choiceTeamSet);
                
                log.debug("[Evaluation] Track {} Stage {}: {} 팀 합격", 
                         trackNo, stageInfo.getStageNo(), 
                         choiceTeamSet != null ? choiceTeamSet.size() : 0);
                
            } catch (Exception e) {
                // 특정 Stage 조회 실패해도 나머지는 계속
                log.warn("[Evaluation] Track {} 단계 조회 실패: {}", 
                        trackNo, e.getMessage());
                choiceTeamSets.add(null);
            }
        }
        
        // ──────────────────────────────────────────────────────
        // [Step 4] 팀 × 단계(서류/발표/최종) 2중 루프
        //          Set.contains로 O(1) 조회
        // ──────────────────────────────────────────────────────
        
        for (Long teamNo : teamNos) {
            String documentResult = null;
            String presentationResult = null;
            String finalResult = null;
            
            // ⚠️ 단계별로 인덱스가 고정 (운영 규칙)
            // 0=서류, 1=발표, 2=최종
            
            // [서류 단계]
            if (choiceTeamSets.size() > 0 && choiceTeamSets.get(0) != null) {
                // Set.contains = O(1)
                documentResult = choiceTeamSets.get(0).contains(teamNo) 
                    ? "O" : "X";
            }
            // else documentResult = null (이 단계 없음)
            
            // [발표 단계]
            if (choiceTeamSets.size() > 1 && choiceTeamSets.get(1) != null) {
                presentationResult = choiceTeamSets.get(1).contains(teamNo)
                    ? "합격" : "불합격";
            }
            
            // [최종 단계]
            if (choiceTeamSets.size() > 2 && choiceTeamSets.get(2) != null) {
                finalResult = choiceTeamSets.get(2).contains(teamNo)
                    ? "합격" : "불합격";
            }
            
            // 결과 저장
            resultMap.put(teamNo, EvaluationResultVO.builder()
                .documentResult(documentResult)
                .presentationResult(presentationResult)
                .finalResult(finalResult)
                .build());
        }
    }
    
    return resultMap;
}
```

### 3-State 모델 (Optional + String)

```java
public class EvaluationResultVO {
    
    /**
     * null: 해당 단계가 없음 (트랙에서 서류만 있는 경우 발표는 null)
     * "O":  서류 진출 (단계 통과)
     * "X":  서류 미진출 (단계 미통과)
     */
    private String documentResult;
    
    /**
     * null: 발표 단계 없음
     * "합격":   발표 합격
     * "불합격": 발표 불합격
     */
    private String presentationResult;
    
    /**
     * null: 최종 단계 없음
     * "합격":   최종 합격
     * "불합격": 최종 불합격
     */
    private String finalResult;
}
```

## ⚠️ 마주친 문제점 & 해결

### **Problem 1: N+1 쿼리 폭발**

**증상:**
```java
// 팀마다 조회
for (team : teams) {
    for (track : tracks) {
        for (stage : stages) {
            // stage_choice 조회
            queryService.getChoice(track, stage);
        }
    }
}
// 500팀 × 5트랙 × 3단계 = 7500쿼리 ❌
```

**해결:**
```java
// Stage 메타는 1회
allStageInfoList = mapper.getStageInfoByTrack();

// 트랙별 합격팀 Set은 트랙 × 단계만
for (track : tracks) {
    for (stage : stages) {
        choiceTeamSet = queryService.getChoice(track, stage);
    }
}
// 5트랙 × 3단계 = 15쿼리

// 팀 조회는 Set.contains (메모리)
for (team : teams) {
    boolean passed = choiceTeamSet.contains(team);  // O(1)
}
```

### **Problem 2: 합격/불합격/없음 구분**

**증상:**
```
발표 단계가 없는 트랙에서 팀 A
├─ DB: presentationResult = NULL
├─ 앱: "불합격"인가? "없는 단계"인가? 혼동 ❌
```

**해결:**
```java
// null: "이 단계 없음"
if (choiceTeamSets.get(1) == null) {
    presentationResult = null;  // ← 명시적으로 구분
}
// 아니면 존재하는 단계만 조회
else if (choiceTeamSet.contains(teamNo)) {
    presentationResult = "합격";
} else {
    presentationResult = "불합격";
}
```

### **Problem 3: Set vs List 성능**

**증상:**
```java
// List로 하면
for (team : choiceList) {  // O(n)
    if (team.getTeamNo() == targetTeamNo) found = true;
}
// 500팀 × 500단계 = 250,000번 선형 탐색 ❌
```

**해결:**
```java
// Set으로 변환
Set<Long> choiceTeamSet = choiceList.stream()
    .map(GetU300ChoiceTeamResVO::getTeamNo)
    .collect(Collectors.toSet());

// O(1) 해시 조회
if (choiceTeamSet.contains(teamNo)) {  // ← 상수 시간!
```

## 📊 성과

### Before (N+1 폭발)

```
쿼리 수:
├─ Stage 메타: 5쿼리
├─ 팀별 단계별 조회: 500팀 × 3단계 = 1500쿼리
└─ 총: 1505+ 쿼리 ❌

응답시간:
└─ 10~15초 ❌

메모리:
└─ List로 선형 탐색
```

### After (Set Aggregation)

```
쿼리 수:
├─ Stage 메타: 1쿼리
├─ 트랙별 단계별 조회: 5트랙 × 3단계 = 15쿼리
└─ 총: 16쿼리 ✅

응답시간:
└─ 200~300ms ✅

메모리:
└─ Set.contains = O(1)
```

### 성능 개선

```
쿼리: 1505 → 16 (99% 감소!)
응답: 15초 → 300ms (50배 개선!)
```

## 🎓 배운 점

### 1. Set은 멤버십 검사의 최강자

> **"contains 조회를 많이 하면 Set을 쓰자"**

```
List.contains = O(n)
Set.contains = O(1)

500팀 조회 시:
└─ List: 500번 순회
└─ Set: 1번 해시
```

### 2. 메타 데이터는 최소 1회

> **"공통으로 쓰는 데이터는 한 번만 로드"**

```java
// ❌ 팀마다 로드
for (team : teams) {
    allStages = queryService.getStages();  // 반복!
}

// ✅ 한 번만 로드
allStages = queryService.getStages();
for (team : teams) {
    // allStages 재사용
}
```

### 3. 3-State는 비즈니스 로직을 명확히

> **"없음 ≠ 불합격"을 코드로 표현하자**

```
null:  단계 없음
"O":   합격
"X":   불합격
```
